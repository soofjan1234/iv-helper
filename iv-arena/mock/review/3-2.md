### 第 1 题：Go 的优点是什么？
**评分**: 知识 5.0 | 表达 4.0 => **A (4.5)**

**你的回答（大意）**: 天然支持并发、goroutine、GMP 调度；概念清楚简单易学；性能高、编译成二进制便于部署、无需虚拟机；支持高效的并发垃圾回收。

**分析**:
- ✅ 并发、GMP、易学、编译型、GC 都说到，结构清晰。

**复习建议**: 可固定一句收尾（如「部署简单、适合云原生」），避免临场想词。

### 第 2 题：make 和 new 的区别？各自适用场景？
**评分**: 知识 5.0 | 表达 4.0 => **A (4.5)**

**你的回答（大意）**: 适用对象不同：new 适用所有类型，make 只用于 slice、map、channel；make 可指定长度和容量；new 返回指针并赋零值。

**分析**:
- ✅ 对象差异、make 容量、new 指针零值都对。

**复习建议**: 保持即可。

### 第 3 题：sync.WaitGroup、sync.ErrGroup 的用途？
**评分**: 知识 3.0 | 表达 3.5 => **B- (3.25)**

**你的回答（大意）**: WaitGroup 用于批量发 HTTP 请求；ErrGroup 包装了 WaitGroup，加了错误捕获和取消，收到错误时可以取消其他请求。

**分析**:
- ⚠️ **纠偏**：WaitGroup 的用途是「等待一组 goroutine 完成」，不限于 HTTP；批量发 HTTP 只是常见用法之一。说「批量 HTTP」容易让面试官觉得你只记得场景而不知道本质。
- ✅ ErrGroup = WaitGroup + 错误聚合 + context 取消，表述正确。

**复习建议**: 先说「WaitGroup 用来等待一组 goroutine 都完成」，再举「比如批量发 HTTP 请求时用」。

### 第 4 题：讲讲 G、M、P 的关系？
**评分**: 知识 3.5 | 表达 3.5 => **B (3.5)**

**你的回答（大意）**: G 是协程，M 是线程，P 是处理器负责调度 G；每个 P 持有一个 M，每个 M 持有各自的 P，每个 P 持有一个 G 的队列。

**分析**:
- ⚠️ **纠偏**：更准确的说法是：**P 是逻辑处理器，有本地 G 队列**；**M 是工作线程，绑定到 P 上执行 G**。关系是「P 上有 G 队列，M 从 P 上取 G 跑」，而不是「每个 P 持有一个 M」（M 和 P 是 M:N，多个 M 可轮流跑在不同 P 上）。
- ✅ G/M/P 三个名字和「调度」作用说对了。

**复习建议**: 背一句「P 维护本地 runqueue，M 绑定 P 后从该 queue 取 G 执行；多个 M 可在不同 P 之间迁移」。

- ✅ 栈/堆存什么、特点、开销都到位。

**复习建议**: 保持即可。

### 第 7 题：Binlog、redolog、undolog 区别？
**评分**: 知识 4.0 | 表达 4.0 => **B (4.0)**

**你的回答（大意）**: Binlog 是 MySQL 层，redolog/undolog 是 InnoDB 层；Binlog 是二进制日志，redolog/undolog 有逻辑（redolog 记哪一页改了啥，undolog 记数据原来怎样、操作是什么）；Binlog 用于主从复制，redolog 用于崩溃恢复，undolog 用于事务（回滚/MVCC）。

**分析**:
- ✅ 层级、内容、用途都对；若追问可补「undolog 还支撑 MVCC 快照读」。

**复习建议**: 保持即可。

### 第 8 题：为什么用 B+ 树？和其他方式对比？
**评分**: 知识 5.0 | 表达 4.0 => **B (4.5)**

**你的回答（大意）**: Hash 查找快但有冲突；二叉查找树易退化成链表；平衡树/红黑树要维护平衡，且对磁盘 I/O、局部性有要求；B+ 树高度低（千万级三四层）、叶子节点双向链表便于范围查询。

**分析**:
- ✅ Hash/二叉/平衡树/红黑树/B+ 的优缺点和 B+ 的 range 查询都说到。

**复习建议**: 保持即可。

### 第 9 题：锁的两种层面是什么？
**评分**: 知识 3.5 | 表达 4.0 => **B (3.75)**

**你的回答（大意）**: 一种是读锁，一种是写锁。

**分析**:
- ⚠️ 题目若指「层面」多为 **表级锁 vs 行级锁**（或「锁的粒度」）。读锁/写锁一般是 **共享锁 S vs 排他锁 X**（锁的模式）。你答的读/写锁在「锁模式」上是对的，但若面试官想问「两种层面」是表 vs 行，会以为你没区分清楚。

**复习建议**: 背两套说法都准备——（1）按粒度：表级锁、行级锁；（2）按模式：共享锁 S（读锁）、排他锁 X（写锁）。被问到「两种层面」先答「表级和行级」，再补「按模式还有共享锁和排他锁」。

### 第 10 题：事务的 ACID？
**评分**: 知识 4.0 | 表达 3.5 => **B (3.75)**

**你的回答（大意）**: A 原子性，事务要么全完成要么全失败；C 一致性，事务前后数据保持一致；I 隔离性，事务之间互不干扰；D 持久性，事务的变更持久有效。

**分析**:
- ✅ 四条都说到；C 可再背一句「约束、业务规则在事务前后都成立」更稳。

**复习建议**: 保持；若想加强可补一句一致性的标准表述。
