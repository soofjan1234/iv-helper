## 索引组织表（Index Organized Table）

意思是按主键顺序组织存放。如果没有主键，先去看是否有非空的唯一索引，否则自动创建一个6字节大小的指针

## 逻辑存储结构

**表空间 (Tablespace) → 段 (Segment) → 区 (Extent) → 页 (Page) → 行 (Row)**

1. 表空间。对应磁盘上的一个或多个物理文件（如 `customer.ibd`）
2. 段。不是一个连续的物理区域，而是一个**逻辑概念，比如：**
    - **数据段（Leaf node segment）**：存 B+ 树的叶子节点（真正的数据）。
    - **索引段（Non-leaf node segment）**：存 B+ 树的非叶子节点（目录）。
    - **回滚段（Rollback segment）**：存用于事务回滚的数据
3. 区。是**连续页组成**的一段空间。
    - **规格**：在默认页大小为 16KB 的情况下，**1 个区 = 64 个连续的页 = 1MB**。
    - **为什么存在**：为了**顺序 I/O**。
        - 当段需要增长时，InnoDB 不再一个页一个页地申请，而是**一次申请 4~5 个区**。这样磁头在读取大量数据时，可以顺着物理轨道一口气扫过去，不用跳来跳去。
4. 页。InnoDB **磁盘管理的最小单位**
5. 行。存储真实的记录字段

## 行记录格式

一行记录长什么样？

1. 变长字段长度列表。针对varchar、text等字段，每个字段占多少字节，逆序存放
2. Null值列表。标记那些字段是Null
3. 记录头信息。
    - 删除标记
    - 最小事务ID
    - 下一条记录的指针（形成链表）
    - 记录类型（普通、删除、最小、最大）
4. 真实列数据
5. 隐藏列
    - DB_ROW_ID：没主键时用的行ID
    - DB_TRX_ID：事务ID
    - DB_ROLL_PTR：回滚指针（undo 位置

### 1. Compact vs Redundant

这两个格式决定了“行”的头部是怎么长的。

- **Redundant（冗余格式）**：这是 MySQL 5.0 之前的旧格式。它很耿直，字段长度等信息存得比较死板，占空间大。现在基本只为了兼容性存在。
- **Compact（紧凑格式）**：从 5.1 开始引入。它的核心思想是能省则省。比如说变长字段长度列表，NULL标志位。

### 2. Dynamic 与 Compressed

现在 MySQL 5.7 和 8.0 默认基本都是 **Dynamic**。

- **Dynamic（动态格式）**：Compact 的升级版。在处理“行溢出”时比 Compact 更聪明（完全溢出），能让索引页承载更多的行，从而让 B+ 树更矮更胖。
- **Compressed（压缩格式）**：在 Dynamic 的基础上，会对页进行 **zlib 算法压缩**。它能省磁盘，但代价是 CPU 压力大，因为读写都要解压/压缩。

### 行溢出

默认页大小的16kb，又一页至少两行，每行就接近8kb，像变长字段就会被踢出去，比如：varchar、text、blob。

Compact和Redundant是留个脑袋，身子扔出去。768 字节后面跟着一个 20 字节的**指针**，指向数据真正存放的“溢出页“。

Dynamic和Compressed只留指针。

**溢出页（Off-page）**通过**链表**连接

## 数据页结构

1. File Header（文件头，固定38字节）
    1. 页的全局身份证
    2. 让InnoDB知道这页在哪、属于谁、前后页是谁。
2. Page Header（页头，固定56字节）
    1. 这个页内部状态
    2. 管理这一页的运行状态，比如有多少行，最后插入的位置在哪
3. Infimum + Supremum（虚拟行）
    1. 每一页固定有两行假数据
    2. Infimum：最小值，Supremum：最大值
    3. 所有真实行都在它们之间，方便形成链表、判断边界。
4. User Records（真正存行的地方）
    1. 就是前面的：行格式（变长列表 + NULL列表 + 头 + 真实数据）
    2. 这些行在页里是单向链表：Infimum → 行1 → 行2 → 行3 → Supremum
5. Free Space（空闲空间）
    1. 新插入的数据从这里拿空间
    2. 用完了就只能分裂页
6. Page Directory（页目录）
    1. 一页里快速查找
    2. B+树树定位到页，页加载到内存，再通过page directory进行二叉查找
7.  File Trailer（文件尾，固定8字节）
    1. 作用：判断这个页是不是写坏了（掉电、损坏）
